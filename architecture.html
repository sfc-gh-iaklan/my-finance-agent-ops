03Semantic View Audit Engine

Static analysis of semantic view definitions — runs without a Snowflake connection. Parses both YAML and SQL DDL formats. Blocks deployment on CRITICAL or ERROR findings (CD only; CI reports findings to the PR without blocking).

📝 Documentation

Checks that all columns, metrics, and tables have descriptions. Short descriptions (<10 chars) flagged as INFO.

  • Missing COMMENT on column → WARNING
  • Missing COMMENT on metric → WARNING
  • Description too short → INFO

🏷️ Naming Conventions

Ensures consistent casing and no special characters in logical names and aliases.

  • Special chars in table name → WARNING
  • Inconsistent alias casing → INFO

📊 Metadata Completeness

Validates categorical columns have sample_values (or VALUES) and tables define at least one metric.

  • Categorical column without VALUES → WARNING
  • Table with no metrics defined → INFO

🔗 Relationships

Verifies relationship coverage for multi-table views and validates table references.

  • No relationships across N tables → ERROR
  • Insufficient relationships → WARNING
  • Unknown table in relationship → ERROR

⚠️ Inconsistencies

Detects conflicting definitions across tables — duplicate metrics or filters with different expressions.

  • Duplicate metric names → HIGH
  • Conflicting filter definitions → CRITICAL

🔄 Duplicates

Finds redundant descriptions that could confuse Cortex Analyst during column disambiguation.

  • Same description on two columns → MEDIUM

Severity Levels

SeverityBlocks Deployment?Description
CRITICAL⛔ Yes — blocks deployConflicting definitions that will produce wrong results
ERROR⛔ Yes — blocks deployMissing essential structure (relationships, references)
HIGHNoDuplicate definitions that degrade quality
WARNINGNoMissing best practices (documentation, metadata)
MEDIUMNoRedundancies that reduce clarity
INFONoSuggestions for improvement

Dual Format Support

YAML (Recommended)

# Parsed via parse_yaml()
tables:
  - name: PRODUCTS
    dimensions:
      - name: category
        sample_values:
          - Electronics
          - Clothing
        is_enum: true
    facts:
      - name: unit_price
    metrics:
      - name: total_revenue
        expr: SUM(unit_price * quantity)

SQL DDL (Legacy)

-- Parsed via parse_ddl()
CREATE SEMANTIC VIEW my_sv
  RETAIL_AI_DEV.ANALYTICS.PRODUCTS
    AS PRODUCTS PRIMARY KEY(PRODUCT_ID)
  WITH COLUMNS (
    CATEGORY AS "Category"
      COMMENT 'Product category'
      VALUES('Electronics','Clothing')
  )
  WITH METRICS (
    "Total Revenue" AS
      SUM(UNIT_PRICE * QUANTITY)
  )

04Evaluation Framework

Two complementary evaluation approaches — question bank batch evaluation and native Snowflake AI evaluation — ensure both functional accuracy and safety.

Semantic View Evaluation

CategoryQtyMethodPROD Threshold
Easy10SQL comparison + LLM judge95%
Hard10SQL comparison + LLM judge75%
Ambiguous10LLM-as-a-Judge only60%

Each question includes a ground-truth SQL query. The framework compares Cortex Analyst's generated SQL results against the expected results. Ambiguous questions use an LLM judge for semantic equivalence.

Alert Thresholds

👎 Negative Feedback Spike

  • WARNING: >25% negative (min 5 responses)
  • CRITICAL: >50% negative

📉 Accuracy Regression

  • WARNING: >10% drop between runs
  • CRITICAL: >20% drop

🐢 Latency Degradation

  • WARNING: P95 > 30 seconds
  • CRITICAL: P95 > 60 seconds

💰 Cost Anomaly

  • WARNING: Daily > 2x 7-day average
  • CRITICAL: Daily > 5x average

🔴 Error Spike

  • WARNING: Error rate >10% (min 10 reqs)
  • CRITICAL: Error rate >25%

🏥 Health Failure

  • CRITICAL: Any health check UNHEALTHY
  • 7 checks: SV, agent, analyst, freshness, errors, alerts

07Interaction Quality Rules Engine

Rules-based detection over snowflake.local.ai_observability_events — identifies problematic agent interactions without requiring an LLM. Runs daily at 02:30 UTC.

Request-Level Flags

FlagRuleSeverity
Tool LoopingSame tool called 3+ timesWARNING
Excessive Steps4+ planning stepsWARNING
Slow RequestDuration > 60 secondsWARNING
High Token Burn> 100k tokensWARNING
Planning ErrorAny step status = ERRORCRITICAL

Thread-Level Flags

FlagRuleSeverity
Abandoned3+ turns, no follow-up 30minWARNING
Rapid Rephrasing3+ turns in <5 minutesWARNING

Alert fires when >20% of requests are flagged or any CRITICAL flags exist in the scan window.
